# 後台管理系統頁面結構 (Admin Web Structure)

基於 Mobile App 的功能需求，轉化為後台管理模組。
策略：Mobile 提供「數據呈現」，Admin 提供「數據管理」。

```mermaid

    SystemMgr --> AppConfig
    SystemMgr --> Logs
```

## 設計策略 (Design Strategy)
由於只有 App UI 參考圖，後台設計將遵循以下原則：
1.  **配色 (Color)**: 提取 App 的主色調（通常是 Logo 色或頂部導航色）作為後台的主題色 (Primary Color)。
2.  **數據對應 (Mapping)**:
    - App `Posts` -> Admin `內容審核`
    - App `Profile` -> Admin `用戶列表`
    - App `Block` -> Admin `黑名單管理`
3.  **UI 框架**: 建議使用成熟的 **Ant Design** 或 **MUI**，以快速搭建標準化的表格與表單，專注於功能邏輯。
